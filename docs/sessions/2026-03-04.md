### Decisions
- **Sparkle plist keys via Info.plist file**: The `INFOPLIST_KEY_` build setting prefix only injects Apple-recognized keys into generated plists. Custom keys (SUFeedURL, SUPublicEDKey) must go in a real Info.plist file referenced via `INFOPLIST_FILE`. Xcode merges both.
